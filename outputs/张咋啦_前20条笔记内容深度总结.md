## 二、前20条笔记内容深度总结

### 1. HTML Slides 教程（最高互动笔记之一）
**核心内容**：张咋啦开源了 Frontend Slides 项目，让非程序员也能用 AI 自动生成美观的 HTML 幻灯片。教程分为三步：①安装 SKILL.md 到 Claude Code 或 WorkBuddy；②用自然语言描述需求（如"帮我做一份关于XX主题的PPT"）；③AI 会先采访你了解审美偏好，然后生成 3 个风格预览供选择。她强调"展示而非讲述"——AI 不问你想要什么配色，而是直接给出视觉方案。该工具支持嵌入推文、视频、转场动画，5-10 分钟即可完成演示文稿。

### 2. LongCut.ai 产品介绍
**核心内容**：介绍自己开发的 LongCut.ai 产品，定位为"深度学习长视频工具"。用户上传 YouTube 视频后，AI 会自动分段总结（按访谈主题）、实时语音转字幕（支持中英日三语）、并提供 AI Chatbot 随时提问。她特别强调这不是一个简单的播放器——它会用颜色高亮关键内容，点击字幕可直接跳转到对应时间点。产品核心理念是"Top Down Learning"：先快速浏览全貌，再选择深入探索的部分，而不是从头到尾被动观看。

### 3. Vibe Coding 核心理念（7条原则）
**核心内容**：她提出了 Vibe Coding 的 7 条核心原则：①Build first, learn later（先做再学）；②Show, not tell（直接展示原型而非空谈）；③Tailor your learning（用自己的方式消化知识，如把视频转电子书）；④Push until it breaks（试探 AI 的能力边界）；⑤Try everything, waste tokens（多尝试不怕浪费 token）；⑥Build in public（边学边做边公开）；⑦Make it fun（保持乐趣）。她认为在 AI 时代，Vibe Coding 的本质是让 AI 负责代码实现，人专注于创意和产品打磨。

### 4. 从非技术到 AI 产品经理的个人路径
**核心内容**：详细讲述了自己的职业转型历程：①哈佛大学心理学本科 → ②硅谷科技媒体 The Information 驻华记者实习 → ③GGV 纪源资本投资分析师（主持播客《996》采访李开复、张一鸣等）→ ④字节跳动国际化战略 → ⑤飞书产品营销负责人 → ⑥AI 产品经理。她特别强调，自己在 GGV 期间做播客的经历是她最大的财富——"不是人脉，而是学会了如何系统地采访和思考"。

### 5. Build in Public 方法论详解
**核心内容**：详细讲解了她的 Build in Public 方法论。她提出"定位不是想出来的，是做出来的"——她连续日更半年一天不间断，通过持续输出和反馈才锁定"非技术背景分享 AI 趋势 + 个人成长"的定位。具体策略：①冷启动选题：回想常被问的 3 个问题，这就是你擅长且有价值的起点；②内容标准：follow the builders not influencers——只推一手洞察，每篇加自己观察；③重复是策略：相同选题换侧重点重复 100 遍都行，因为传播有损耗。

### 6. 内容创作方法论（定位+标准+涨粉）
**核心内容**：分享了她做自媒体的完整方法论。关于定位："想不出方向就想 3 个朋友常问你的问题，这就是你擅长且有价值的起点"。关于内容标准："Follow the builders, not influencers——只推 AI 创始人、从业者的一手洞察，拒绝营销号带节奏。每篇必加自己的观察和见解，这是差异化关键"。关于涨粉策略："量变引起质变，500+ 篇内容，平均一天 1.5 篇，连续日更半年不中断"。

### 7. 个人品牌核心理念
**核心内容**：她的个人品牌由几个核心观点构成：①Build in Public——不要害怕别人知道你在学什么、做什么，分享学习、实践和踩坑的过程；②Learn in Public——公开构建成长而非展示成长，为了持续输出倒逼自己高质量输入；③松弛感的成长——找到那些"在你看来像玩，在别人看来像工作"的事情；④Code 和 Media 是双重杠杆，不需要任何人许可就能产生复利；⑤AI 时代是文科生最好的时代，讲故事、共情力、"活人感"成为稀缺能力。

### 8. Vibe Coding 实操 7 步框架
**核心内容**：详细的 Vibe Coding 入门路径：①选对起点项目（纯前端网站、个人简历站、作品集页，不做复杂后端应用）；②用对工具（Faces、Replit Design 自然语言驱动的平台，CleanShot X 录屏工具）；③掌握设计语言（了解 Pixel Style、Neo-brutalism 等术语，直接截图告诉 AI"做一个这样的"）；④迭代式开发（搭乐高思维，不要期望一次 prompt 就完美）；⑤高效 Prompt 技巧（"一次问我一个问题"让 AI 像产品经理一样引导你）；⑥了解可调用的 API（知道工具存在即可，让 AI 去调用）；⑦正确的学习心态（边做边学，而非先学完再做）。

### 9. 产品发布策略（三层模型）
**核心内容**：她从原型到影响力的发展模型：①Prototype 阶段——先录 demo 视频发到社交媒体，用转赞评验证需求是否真实，即使产品还没上线也要先发视频；②Product 阶段——视频反响好再投入资源做稳定服务，用视频数据说服专业人士合作；③Business 阶段——side project 暂时别想商业化，会失去乐趣。她还提出被抄袭的正确认知："99% 的情况你发布后根本没人看，真正的护城河是 Distribution（社交媒体粉丝、个人品牌、流量渠道）"。

### 10. 飞书 CLI + Claude Code 自动化工作流
**核心内容**：介绍了 lark-cli 的安装和使用：①安装 npm install -g @larksuite/cli；②添加技能包；③初始化配置并登录授权；④在飞书群内使用指令如 `/new chat`、`/cd [路径]`、`新增 Todo: [任务]`。她分享了两大方案：一是飞书机器人直连 Claude Code，群里@机器人即可调用本地编程助手；二是极客待办神器——飞书发消息自动同步到本地 Obsidian Markdown 文档。核心理念是"让 AI 真正走进你的工作环境"。

### 11. 产品思维的三个核心维度
**核心内容**：她提出产品经理的三个核心思维：①内容与产品相互反哺——产品通过内容传播获得种子用户，用户反馈又塑造产品迭代方向；②Learning and Building in Public——公开学习实践能扩大验证产品价值的样本，避免在极小样本测试下出现需求判断偏差；③打破学科标签——顶级产品经理都是感性与理性的综合体，不要用文科生/理科生定义自己。AI 时代的技术实现变廉价后，文科生在产品设计、需求洞察上反而有先发优势。

### 12. 为什么 HTML Slides 能获得 17k+ Stars
**核心内容**：解释了为什么 Frontend Slides 能在 GitHub 大火：①解决了真实痛点——传统 PPT 制作耗时耗力，AI 自动生成网页级幻灯片只需要 5 分钟；②内置 12 种精选主题，专门避开"AI 味"紫色渐变审美；③"先看后选"的工作流——AI 先问你想要什么感觉，生成 3 个视觉预览让你挑，不用你描述设计偏好；④支持 PPT 转网页，自动提取文字、图片、笔记；⑤Skill 架构精妙——主文件只有 183 行当"目录"，12 个预设主题可自由切换。

### 13. 用 Cursor/GPT 做原型的可视化沟通法
**核心内容**：她强调"技术素养"（Technical Literacy）——不是让你自己写代码，而是要知道技术能实现什么效果。她用 Cursor、GPT 等工具快速做出原型的方法：先理解用户需求 → 录可视化 Demo → 用工具做出 MVP → 让团队直观理解产品意图。她说："真正拉开差距的不是你会不会写 React，而是你能不能用最快方式让团队看到你想表达的产品形态"。

### 14. Agency（能动性）：像创始人一样行动
**核心内容**：讨论什么是"Agency"——像创始人一样思考与行动。她提出把目标拆解为可执行动作，克服障碍并交付结果，从"完成任务"转向"创造价值"。具体做法：①不要等完美时机，先开始再迭代；②遇到问题主动寻找方案，而非向上汇报困难；③把每次交付都当成作品而非任务。她认为这是非技术背景的人进入 AI 行业最重要的软技能。

### 15. AI 时代的技能学习法：Tailor Your Learning
**核心内容**：她提出每个人应该定制自己的学习方式。有人把视频转电子书学习，有人把数据转可视化看板，有人用闪卡记忆。关键是"用自己的方式消化知识"，而不是模仿别人。她分享了自己的方法：做播客采访 → 听嘉宾回答 → 写笔记总结 → 发社交媒体 → 观众反馈 → 形成新选题。这是一个"输入-输出-反馈-再输入"的闭环。

### 16. 冷启动涨粉的真实经历
**核心内容**：她坦承自己犹豫了一年多才开始发小红书，期间买了各种设备（灯、三脚架等）但一直没开始，设备都落灰了。最后的突破点是"不管多粗糙先发出来"——用手机开始讲大家经常问她的实际问题。前几篇笔记互动量很低，但她坚持日更，半年后才慢慢形成正反馈循环。她说："观众觉得你的定位才是你的定位——不是你自己想的"。

### 17. 被质疑与回击：关于 Vibe Coding 的争议
**核心内容**：有人质疑"非程序员做软件一知半解瞎带节奏"，认为软件工程很复杂不是 Idea 就能做好的。她的回应是：①她不否认软件的复杂性；②Vibe Coding 的价值不在于替代专业开发，而在于让非技术背景的人也能把想法变成可运行的原型；③"Coding 便宜了，但人们对软件的期待也更高了"——这正是需要更多跨学科人才的原因。

### 18. 独立开发者的定价思考
**核心内容**：分享了一个具体案例——把自己的工具定价从 99 改到 199。她详细解释了思考过程：考虑变量包括成本、竞品价格、目标用户付费意愿、自我价值感。她说："做定价决策的时候暴露思考过程，让用户看到真实的人面对真实选择时的判断，这本身就是一种 Build in Public"。这个案例展示了如何用决策透明化建立用户信任。

### 19. 避免"创业者回音室"陷阱
**核心内容**：她提醒 Build in Public 的创业者要注意：公开构建容易吸引同行、工具爱好者和创业观光客，他们的反馈可能与真实用户需求存在偏差。她提出了"定向透明策略"：想法验证期不问"创意如何"而问"愿为解决方案付多少钱"；原型测试期用预购数据而非口头认可验证需求；只在关键节点选择性公开进展。

### 20. 给非技术背景的三条建议
**核心内容**：总结了她给非技术背景从业者的三条核心建议：①用"提问"代替"代码"——去离技术最近的地方做观察者，训练信息检索能力、深度思考能力和表达能力，这些在 AI 工具普及的今天变成核心生产力；②杠杆思维——写作是杠杆，播客是杠杆，AI 更是杠杆。找到"对你来说像玩，但对别人来说是工作"的事情；③不要在建立自己的审美之前让 AI 替你完成本该属于你的思考——真正吸引人的是真实、质朴、有观点的人，不是 AI 生成的完美模板。
